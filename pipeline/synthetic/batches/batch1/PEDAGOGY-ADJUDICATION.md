# Batch 1 — pedagogy adjudication

Seven units reviewed by skill-guided reviewers (`pedagogy-review`, Opus) for comprehension integrity, distractor pedagogical value, **rationale factual accuracy**, and construct validity (surface tells) — the dimensions the mechanical gates and the language pass do not cover. Fixes applied and re-verified (blind G-KEY ×2 + G-DISTRACTOR): keys unchanged, no new double-keys; `candidates-final/` is the fully-adjudicated batch.

**Verdicts:** 3 SOUND · 4 MINOR_FIXES · 0 NEEDS_REDESIGN · 0 REJECT. Nothing mis-teaches after fixes.

> **What this layer caught that nothing upstream did:** (1) `elf-b1-002`'s rationales — the teaching text — stated **false** collocation facts ("dilute doesn't take margins"; it does). (2) A **systemic** construct-validity leak on `las-b1-001` and `elf-b1-001`: because the traps are overgeneralisation/quantifier-upgrade, the key was always the lone *hedged* option, so "pick the qualified answer" scored blind. Fixed at the source (M-TELL gate + a GENERATION.md law), and rebalanced in these units.

## elf-b1-002 · ELF — **MINOR_FIXES**

**Findings & fixes:**

- **[major/rationale]** q4 / gap 4 — «dilute = collocation_misfit (dilute collocates with shares/brand, not margins)»  
  → **fix:** Replace the false reason with the true discriminator: 'dilute margins' / 'margin dilution' / 'dilutive to margins' IS an attested business collocation, so the rejection is not about collocation. It fails on sense/mechanism: dilution means lowering an AVERAGE margin by mixing in lower-margin lines, not money lost to dead stock; 'erode margins' is the idiom for profitability eaten away by cost, and 'overnight' wants a decisive-loss verb. Keep key 'erode' and keep 'dilute' as a genuine near-miss distractor.  
  *why:* The rationale rejects 'dilute' for a demonstrably false collocation fact (web-verified: margin dilution is standard finance usage), teaching a false fact exactly like the Batch-1 cloze defect; key and distractor text are unchanged and 'erode' stays uniquely best, so no G-KEY/G-DISTRACTOR re-gate is needed.

- **[minor/rationale]** q2 / gap 2 — «familiarity = collocation_misfit (does not take 'has quietly deepened' here)»  
  → **fix:** State the real reason: 'familiarity' does collocate with 'deepen' (deepening familiarity is attested), so this is not a collocation ban. It fails on referent/possessive — 'their familiarity' would be the cafes' own familiarity, but the growing draw is the public's familiarity WITH them; 'popularity' is precisely what waiting lists and courting publishers evidence.  
  *why:* 'familiarity deepened' is idiomatic English, so the stated collocation-misfit reason is false; the true failure is a possessive/referent mismatch, and correcting the prose does not alter key or options.

- **[minor/rationale]** q3 / gap 3 — «hostile = collocation_misfit/sense (company is not an antagonist of downloads)»  
  → **fix:** Drop the 'collocation_misfit' label — 'hostile to X' is a valid collocation — and keep only the sense reason already given: the passage frames the table/company as enduring and resistant, not as an antagonist, so 'hostile to the download' personifies wrongly.  
  *why:* 'hostile to' collocates fine, so the collocation-misfit tag is the same mislabeling seen at Q4/Q2; the sense reason alone is correct and decisive.

- **[minor/distractor]** q1 / gap 1 _(re-gated)_ — «loyalty and casualty are real -ty nouns that produce no idiomatic reading with 'a passing ___'»  
  → **fix:** Optional, low-priority: 'loyalty' is near-filler (no real misconception pull under 'a passing ___, a fad'). If strengthening, swap it for a -ty noun with a genuine lure; the rhyme-set design constrains options, so this is discretionary. 'casualty' can stay — it carries a faint thematic lure via 'fold' (a folded business is a casualty).  
  *why:* The rationale concedes these produce 'no idiomatic reading', i.e. loyalty is close to filler; swapping a distractor touches the option set and would need a G-DISTRACTOR re-check, but the current key stays uniquely correct so the fix is optional.
